01 Discover

诊断现有资料和表达问题

第一步不是立刻设计网页，而是先理解客户当前已有的材料。很多制造业企业并不是没有实力，而是实力没有被正确表达。

A. Input / 客户提供

  • Existing website URL
  • Company profile or brochure
  • Product photos or catalogs
  • Equipment list
  • Project cases
  • Target market

B. What We Check / 我们检查

  • 信息是否清晰易懂
  • 产品分类是否符合买家逻辑
  • 制造核心能力是否被突出
  • 英文表达是否自然专业
  • 是否适合 SEO / AI 阅读

C. Output / 阶段输出

  • Digital asset gap diagnosis
  • Priority problem list
  • Initial service recommendation
  • Service starting point

"The first goal is not to sell a package. The first goal is to find the right starting point."

第一步不是推销套餐，而是找到最适合客户当前阶段的切入点。

02 Analyze

分析客户搜索意图与内容结构

制造业数字资产不是单纯设计问题。页面结构、分类、FAQ和案例表达，都会影响海外买家是否能快速判断你的专业度。

A. Input / 分析输入

  • Main products / 主营产品
  • Buyer questions / 客户问题
  • Target industries / 目标行业
  • Existing keywords / 现有关键词
  • Sales materials / 销售资料

B. What We Analyze / 分析内容

  • Buyer search intent / 搜索意图
  • Content hierarchy / 内容层级
  • Trust proof gaps / 信任背书缺口
  • FAQ opportunities / FAQ机会
  • GEO/AEO readability / AI可读性

C. Output / 阶段输出

  • Content architecture map
  • Keyword / intent direction
  • Page structure recommendation
  • Case / capability positioning

"A good website should reduce explanation cost before the first sales call."

好的制造企业官网，应该在第一次销售沟通之前，先降低客户理解成本。

03 Design

重构网站、文案、视觉和内容资产

设计不仅是“让页面变好看”，而是把企业能力转化成可理解、可搜索的商业资产。视觉、文案和结构必须共同建立信任。

A. What We Design / 设计对象

  • Website pages / 网站页面
  • Homepage structure / 首页结构
  • Product & Case pages / 产品案例页
  • Social media templates / 社媒模板
  • AI prompt workflows / AI工作流

B. Design Principles / 设计原则

  • Clear before fancy / 先清晰再炫技
  • Industrial credibility / 工业可信度
  • Search-readable structure / 搜索友好
  • Reusable content / 可复用模块
  • No fake claims / 不制造虚假承诺

C. Output / 阶段输出

  • Visual direction & Layout
  • Copy refinement
  • CTA system
  • Social & AI draft tools

"The goal is not just better visuals. The goal is better business communication."

目标不只是视觉好看，而是让商业沟通更清楚、更可信。

Service Rules

Clear Scope. Clean Collaboration.

方法论的目的不是让流程变复杂，而是避免传统外包合作中的模糊承诺、无限修改和责任不清。

Scope Matrix & Boundaries
  • 开始前明确界定交付范围、页面数量和内容深度。
  • 开始前明确合理的设计与文案修改轮次。
  • 项目推进需要客户按时提供基础资料并进行内部工程审核。
  • 不承诺短期内（如一个月）的 SEO 关键词排名暴涨。
  • 不承诺具体的询盘数量或销售额度提升保证。
  • 不替代客户内部核心技术参数判断，AI生成内容必须经人工复核。
  • 超出合同约定的额外页面、功能或视频拍摄需求，按范围补充报价。
